Nova

Basic Instructions

Installing the Binaries

The binaries are available on SourceForge in a zip file. If you unzip this file you will get 4 folders Nova/Component Editor, Nova/Console Nova/Race Designer  and  the Nova GUI.   In each folder you will find a .exe or application file. Running it will run the program of the same name. There is no need to install/uninstall the programs.

Note that this version of the Nova binaries require that .Net version 3 is installed. This is a free download from the Microsoft website. If you are using Windows Update you will already have received it automatically.

You may place the Nova folder anywhere you wish, but when you first run Nova you will get an error message because you will need to tell it where you placed it's files. Hopefully a future version will be smart enough to check the obvious places first, but for now you will need to give it a hand. After you do this the first time, Nova will remember where the files are located, by using the windows registry.

Runing the applications in the following order will ensure that the windows registry keys are set for each aplication and should prevent errors.

Component Editor

To start with, open the Nova/Component Editor folder and run the application Component Editor.exe this program is used for browsing (or even modifying) the Nova design components (or technology tree)

Components.xml

Nova needs a file that defines the available design components. By default it's in the top-level folder of the Nova directory and is called components.xml. Select the file and click the Open button.

Graphics

Next Nova needs to know where the Graphics images are located. They should be in Nova/Graphics. Highlight the Graphics folder in the tree view and click the OK button. You should now be able to browse the design components. When you are done with the component editor, close it and move on to making a race.

Create One or More Races

First, the Race Designer must be ran to create a race to play.  Once the race has been designed and the Generate button has been pressed you will be prompted for a folder to hold all files related to the game. The Nova GUI program can currently only play one game at a time and it will automatically use the folder you specify in the Race Designer.

If you want to play a "test" game you can create as many races as you like. You will find things simpler to manage if you select the same folder for all races you create (it will save you copying files around).

Create a Game

Next, run the Console program and press the New Game button. If no folder has been specified before then, again, you will be prompted for one. If you are running a test game then specify the same one given in the Race Designer as it will save the need to copy turn files around.

Create a Player Turn

Then just start the GUI and create a turn. A turn file is created whenever you exit the GUI so there is no explicit save or submit commands as there are in Stars!

Generate a New Year Turn

Then it's a case of starting up the Nova Console and pressing the Generate Turn button to create the game turn for the next year and then re-running the GUI to generate the next player turn. These last two steps are performed until the end of the game.

Game Folders

It is a design aim that all game persistent game data, including that of multiple races, can exist in a single folder on a single PC and accessed by a single user name. This allows a convenient method of debugging the programs (or playing practice games) without having to copy files between folders or log in and out to switch user names to play different races in a game.

There are potentially two folders: the one used for the player's game files (used by the Race Designer and the Nova GUI) and one used by the Nova Console game files.

The first program that must be run is the Race Designer and it will gather (implicitly) the name of the player's game folder when the Generate Race feature is selected and the where to save the race definition file is specified in the Save File dialog . This folder name is saved in the Windows Registry (for the current user) and picked up automatically by the Nova GUI when it starts so it need not be requested from the user.

The Nova Console also needs a game folder to keep its data. It cannot use the one specified by the Race Designer in the Registry as it may, and typically will, be running on a different PC so a folder to use must be explicitly prompted for (using the Windows Browse Folder dialog). However, if the Nova Console is running on the same PC as a game player, the player's game directory can be selected as the one to use. This removes the need to copy turn data files from one place to the other. Once the console game folder has been selected it is saved (in the console state data file) and not asked for again
